## Agent 交互流程

### 审查流水线 (LangGraph)

```
                    ┌──────────────┐
                    │  parse_code  │  解析所有文件 (Tree-sitter AST)
                    └──────┬───────┘
                           │
            ┌──────────────┼──────────────┬──────────────┬──────────────┐
            │              │              │              │              │
     ┌──────▼──────┐ ┌────▼────┐ ┌──────▼──────┐ ┌────▼────┐ ┌──────▼──────┐
     │   style     │ │security │ │architecture │ │perf     │ │  refactor   │
     │   review    │ │ review  │ │   review    │ │review   │ │   review    │
     └──────┬──────┘ └────┬────┘ └──────┬──────┘ └────┬────┘ └──────┬──────┘
            │              │              │              │              │
            └──────────────┴──────────────┴──────────────┴──────────────┘
                                           │
                                  ┌────────▼────────┐
                                  │    arbitrate     │  去重、排序、评分、摘要
                                  └────────┬────────┘

                                           │
                                         END
```

### Agent 内部执行流程

```
┌──────────────────────────────────────────────┐
│              BaseReviewAgent                  │
│                                               │
│  review(parsed_files) → list[dict]            │
│                                               │
│  StyleChecker:                                │
│    1. AST 静态分析 (函数长度、参数数量)        │
│    2. LLM 语义分析 (命名、注释、重复)          │
│                                               │
│  SecurityAuditor:                             │
│    1. 正则模式匹配 (硬编码密钥、SQL注入等)     │
│    2. LLM 深度推理 (XSS、SSRF、路径遍历)       │
│                                               │
│  ArchitectureAnalyzer:                        │
│    1. 依赖图构建 (import 解析)                │
│    2. LLM 架构评估 (模块耦合、分层)            │
│                                               │
│  PerformanceProfiler:                         │
│    1. 圈复杂度计算 (radon)                    │
│    2. AST 嵌套循环/递归检测                   │
│    3. LLM 性能热点分析                        │
│                                               │
│  RefactorAdvisor:                             │
│    1. 代码坏味道检测 (长参数列表、重复代码)    │
│    2. LLM 重构方案生成                        │
│                                               │
│  Arbitrator (特殊 — 不审查代码):              │
│    1. 结果合并与去重                          │
│    2. 冲突消解与严重等级排序                  │
│    3. 评分计算 (扣分制)                       │
│    4. LLM 摘要生成 (中文)                     │
│    5. HTML 报告生成                           │
└──────────────────────────────────────────────┘
```

## 技术决策

| 决策 | 理由 |
|------|------|
| LangGraph 编排 | 图结构状态机，支持并行节点，天然适合审查流水线 |
| LiteLLM 统一调用 | 支持多模型切换 (GLM/DeepSeek/Ollama)，统一接口 |
| Tree-sitter AST | 高性能增量解析，多语言支持，语法树稳定 |
| 四层记忆分离 | 不同生命周期和检索模式，分别选用合适的存储后端 |
| OTLP gRPC | 高性能追踪协议，与 Jaeger/Prometheus/Grafana 生态兼容 |
| Docker Compose | 单机部署简单，服务编排清晰 |
| GitHub Actions | 免费 CI/CD，与 GitHub Webhook 深度集成 |
